SSM整合实现登录验证Demo

环境配置

所具备的环境配置

  • Tomcat
  • JDK12
  • Maven
  • Spring5
  • SpringMVC
  • MyBatis

所用IDE

  • IDEA

创建Webapp项目

  1. IDEA新建项目，选择Maven项目，勾选Create from archetype，选择maven-archetype-webapp
  2. GroupId: com.Lcxq; ArtifactId: SSMLoginDemo
  3. 在maven配置界面选择本地安装的maven目录，并且将setting.xml改为本地maven/conf目录下的setting.xml文件，这样才会使用阿里云镜像实现加速
  4. 最后修改一下需要的项目路径即可，等待maven的自动拉取

创建目录结构

  1. 在main目录下创建java目录，并标记为Sources Root
  2. 在main目录下创建resources目录，并标记为Resources Root
  3. 在根目录下创建target目录，idea将自动识别为Excluded
  4. 在java目录下创建com包，在com包中创建controller,dao,entity,service四个包
  5. 在resources目录下创建mapper包和spring包，以及jdbc.properties文件，log4j.properties文件，mybatis-config.xml文件，用于配置
  6. （如有需要）在src目录下创建test文件夹，并创建java和resources文件夹，并将它们分别标记为Test Resources Root和Test Resources Root

最后目录结构为

├─.idea
├─src
│  └─main
│      ├─java
│      │  └─com
│      │      ├─controller
│      │      ├─dao
│      │      ├─entity
│      │      └─service
│      ├─resources
│      │  ├─mapper
│      │  └─spring
│      └─webapp
│          └─WEB-INF
└─target

maven依赖拉取

在pom.xml中进行如下配置

首先在properties标签中设定spring框架版本和mybatis版本

<properties>
    <spring.version>5.2.6.RELEASE</spring.version>
    <mybatis.version>3.5.2</mybatis.version>
  </properties>

添加spring框架依赖

<!--Spring相关-->
    <dependency>
      <groupId>org.springframework</groupId>
      <artifactId>spring-core</artifactId>
      <version>${spring.version}</version>
    </dependency>
    <dependency>
      <groupId>org.springframework</groupId>
      <artifactId>spring-beans</artifactId>
      <version>${spring.version}</version>
    </dependency>
    <dependency>
      <groupId>org.springframework</groupId>
      <artifactId>spring-context</artifactId>
      <version>${spring.version}</version>
    </dependency>
    <!-- Spring dao依赖 -->
    <dependency>
      <groupId>org.springframework</groupId>
      <artifactId>spring-jdbc</artifactId>
      <version>${spring.version}</version>
    </dependency>
    <dependency>
      <groupId>org.springframework</groupId>
      <artifactId>spring-tx</artifactId>
      <version>${spring.version}</version>
    </dependency>
    <!-- Spring web依赖 -->
    <dependency>
      <groupId>org.springframework</groupId>
      <artifactId>spring-web</artifactId>
      <version>${spring.version}</version>
    </dependency>
    <dependency>
      <groupId>org.springframework</groupId>
      <artifactId>spring-webmvc</artifactId>
      <version>${spring.version}</version>

      <groupId>org.springframework</groupId>
      <artifactId>spring-test</artifactId>
      <version>${spring.version}</version>
    </dependency>
