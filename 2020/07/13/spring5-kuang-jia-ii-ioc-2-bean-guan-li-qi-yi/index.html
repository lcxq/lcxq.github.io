IoC操作之Bean管理

什么是Bean管理（概念）

Bean管理指的是两个操作

  1. Spring创建对象
  2. Spring注入属性

创建对象就是之前入门案例中使用xml配置文件创建对象的方式，这里介绍什么是注入属性

比如依旧是User类，多了一个String类型的userName属性

package com.atguigu.spring5;

public class User {

    private String userName;
    public void add() {
        System.out.println("add.....");
    }
}

现在要实现在创建对象过程中设置userName值。

Bean管理操作有两种方式

  1. 基于xml配置文件方式实现
  2. 基于注解方式实现

先介绍基于xml方式

  1. 基于xml创建对象

    (1) 用bean标签创建，标签里面添加对应属性，就可以实现对象创建

    (2) 在bean标签中有很多属性，介绍常用的属性

    • id属性：给这个对象取一个别名（唯一标识）
    • class属性：类的全路径（包 + 类）
    • name属性（了解一下）：和id类似，现在基本不用了

    (3) 创建对象默认执行无参构造方法

  2. 基于xml方式注入属性

    DI：依赖注入，就是注入属性

  • 第一种方法：使用自定义的set方法注入

(1) 创建类，定义属性，创建属性对应的set方法

package com.atguigu.spring5;

public class Book {
    private String bname;
    private String bauthor;
    public void setBname(String bname) {
        this.bname = bname;
    }

    public void setBauthor(String bauthor) {
        this.bauthor = bauthor;
    }
}

(2) 配置文件

<bean id="book" class="com.atguigu.spring5.Book">
    <!--使用property完成属性注入
        name: 类属性的名称
        value: 向属性注入的值
    -->
    <property name="bname" value="易筋经"></property>
    <property name="bauthor" value="达摩老祖"></property>        
</bean>
  • 第二种方法：有参构造注入

(1) 创建类，定义属性，创建属性对应有参数的构造方法

package com.atguigu.spring5;

public class Orders {
    private String oname;
    private String address;

    public Orders(String oname, String address) {
        this.oname = oname;
        this.address = address;
    }
}

(2) 配置文件

<bean id="orders" class="com.atguigu.spring5.Orders">
    <constructor-arg name="oname" value="abc"></constructor-arg>
    <constructor-arg name="address" value="China"></constructor-arg>
</bean>

和之前不同的就是使用的是constructor-arg标签，也可以用标签里的index属性，但是还是name比较常用

p名称空间注入（了解即可）

使用p名称空间注入，可以简化基于xml方式

xml注入其他类型属性

  1. 字面量

属性的固定值，比如直接在代码中配置 private String name = “abc”; 或者在配置文件中用property标签配置

(1) null值

<property name="address">
   <null/>
</property>

(2) 属性值包含特殊符号

<bean id="book" class="com.atguigu.spring5.Book">
  <!--使用property完成属性注入
      name: 类属性的名称
      value: 向属性注入的值
  -->
  <!--属性值包含特殊符号
      1 把<>进行转义 &lt; &gt;
      2 把带特殊符号内容写到CDATA
  -->
  <property name="bname" value="易筋经"></property>
  <property name="bauthor" value="达摩老祖"></property>
  <property name="address">
      <value><![CDATA[<<南京>>]]></value>
  </property>
</bean>

  1. 注入属性-外部bean

    (1) 创建两个类service类和dao类

    (2) 在service调用dao里面的方法

    (3) 在Spring配置文件中进行配置

<bean id="userService" class="com.atguigu.spring5.service.UserService">
  <!--注入userDao对象
      name属性值：类里面属性名称
      ref属性：创建userDao对象bean标签id值
  -->
  <property name="userDao" ref="userDaoImpl"></property>
</bean>
<bean id ="userDaoImpl" class="com.atguigu.spring5.dao.UserDaoImpl"></bean>

  1. 注入属性-内部bean

    (1) 一对多关系：部门和员工，一个部门有多个员工，一个员工属于一个部门

    (2) 在实体类之间表示一对多关系，员工表示所属部门，使用对象类型属性进行表示

     package com.atguigu.spring5.bean;

 //部门类
 public class Dept {
     private String dname;

     public void setDname(String dname) {
         this.dname = dname;
     }
 }

 package com.atguigu.spring5.bean;

 //员工类
 public class Emp {
     private String ename;
     private String gender;
     //员工属于某一个部门，使用对象形式表示
     private Dept dept;
     public void setDept(Dept dept) {
         this.dept = dept;
     }
     public void setGender(String gender) {
         this.gender = gender;
     }
     public void setEname(String ename) {
         this.ename = ename;
     }
 }

    (3) 进行配置

<!--内部bean-->
 <bean id="emp" class="com.atguigu.spring5.bean.Emp">
     <!--先设置两个普通属性-->
     <property name="ename" value="lucy"></property>
     <property name="gender" value="女"></property>
     <!--设置对象类型属性-->
     <property name="dept">
         <bean id="dept" class="com.atguigu.spring5.bean.Dept">
             <property name="dname" value="安保部"></property>
         </bean>
     </property>
 </bean>

可以看到，对于一个类的属性为另一个类时(区别于外部bean的内部bean)，可以以嵌套的形式进行注入。

  1. 注入属性-级联赋值

(1) 第一种写法

<bean id="emp" class="com.atguigu.spring5.bean.Emp">
  <!--先设置两个普通属性-->
  <property name="ename" value="lucy"></property>
  <property name="gender" value="女"></property>
  <!--级联赋值-->
  <property name="dept" ref="dept"></property>
</bean>
<bean id="dept" class="com.atguigu.spring5.bean.Dept">
  <property name="dname" value="财务部"></property>
</bean>

可以看到，级联赋值其实和外部bean注入十分相似，都是利用ref进行注入

(2) 第二种写法

<bean id="emp" class="com.atguigu.spring5.bean.Emp">
  <!--先设置两个普通属性-->
  <property name="ename" value="lucy"></property>
  <property name="gender" value="女"></property>
  <!--级联赋值-->
  <property name="dept" ref="dept"></property>
  <property name="dept.dname" value="技术部" ></property>
</bean>
<bean id="dept" class="com.atguigu.spring5.bean.Dept">
  <property name="dname" value="财务部"></property>
</bean>

这边关键就是用dept.dname来同时注入内部bean的属性，而且会覆盖掉ref注入的内部bean属性值（财务部->技术部）

注入属性-集合属性

  1. 注入数组类型属性
  2. 注入List集合类型属性
  3. 注入Map集合类型属性

(1) 创建类，定义各种集合类型属性以及对应的set方法

public class Stu {
    // 1 数组类型属性
    private String[] course;
    //2 List集合类型属性
    private List<String> list;
    //3 Map集合类型属性
    private Map<String, String> maps;
    //4 set集合类型属性
    private Set<String> sets;
    public void setCourse(String[] course) {
        this.course = course;
    }
    public void setMaps(Map<String, String> maps) {
        this.maps = maps;
    }
    public void setSets(Set<String> sets) {
        this.sets = sets;
    }
    public void setList(List<String> list) {
        this.list = list;
    }
}

(2) 配置

<!--集合类型属性的注入-->
 <bean id="stu" class="com.atguigu.spring5.collectiontype.Stu">
<!--        1 数组类型属性注入-->
     <property name="course">
         <array>
             <value>java课程</value>
             <value>数据库课程</value>
         </array>
     </property>
<!--        list类型属性注入-->
     <property name="list">
         <list>
             <value>张三</value>
             <value>小三</value>
         </list>
     </property>
<!--        map类型属性注入-->
     <property name="maps">
         <map>
             <entry key="JAVA" value="java"></entry>
             <entry key="PHP" value="php"></entry>
         </map>
     </property>
<!--        set类型属性注入-->
     <property name="sets">
         <set>
             <value>MYSQL</value>
             <value>Redis</value>
         </set>
     </property>
 </bean>

每一种集合类型都用相对应的标签即可，比如对于数组类型，嵌入array标签，数组中的值就用多个value表示。

特别的是，map的标签使用的是entry表示键值对。

  1. 在集合里面设置对象类型的值

我们先自定义一个叫Course的类

public class Course {
    private String cname; //课程名称

    public void setCname(String cname) {
        this.cname = cname;
    }
}

然后Stu类中有一个值类型为Course的List属性，表示这个学生上的一些课。

// 学生所学多门课程
 private List<Course> courseList;
 public void setCourseList(List<Course> courseList) {
     this.courseList = courseList;
 }

下面就进行相应的xml配置

<!--集合类型属性的注入-->
<bean id="stu" class="com.atguigu.spring5.collectiontype.Stu">
<!--        1 数组类型属性注入-->
  <property name="course">
      <array>
          <value>java课程</value>
          <value>数据库课程</value>
      </array>
  </property>
<!--        list类型属性注入-->
  <property name="list">
      <list>
          <value>张三</value>
          <value>小三</value>
      </list>
  </property>
<!--        map类型属性注入-->
  <property name="maps">
      <map>
          <entry key="JAVA" value="java"></entry>
          <entry key="PHP" value="php"></entry>
      </map>
  </property>
<!--        set类型属性注入-->
  <property name="sets">
      <set>
          <value>MYSQL</value>
          <value>Redis</value>
      </set>
  </property>
<!--        注入list集合类型，值是自定义的对象-->
  <property name="courseList">
      <list>
          <ref bean="course1"></ref>
          <ref bean="course2"></ref>
      </list>
  </property>
</bean>
<!--    创建多个course对象-->
<bean id="course1" class="com.atguigu.spring5.collectiontype.Course">
  <property name="cname" value="Spring5框架课程"></property>
</bean>
<bean id="course2" class="com.atguigu.spring5.collectiontype.Course">
  <property name="cname" value="Mybatis框架课程"></property>
</bean>

先在xml的Stu Bean外部定义好需要加入到List中的Course对象，然后再用ref标签将这些Course Bean添加到List中。

  1. 把集合注入部分提取出来公用

(1) 在Spring配置文件中引入名称空间util

<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:util="http://www.springframework.org/schema/util"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
                           http://www.springframework.org/schema/util  http://www.springframework.org/schema/util/spring-util.xsd">
</beans>

(2) 使用util标签完成list集合注入提取

我们创建一个Book类

public class Book {
    private List<String> list;
    public void setList(List<String> list) {
        this.list = list;
    }
}

其中有一个List需要注入

xml进行如下配置

<!--1 提取list集合类型属性的注入-->
<util:list id="bookList">
  <value>易筋经</value>
  <value>九阴真经</value>
  <value>九阳神功</value>
</util:list>

<!--    2 提取list集合类型属性注入-->
<bean id="book" class="com.atguigu.spring5.collectiontype.Book">
  <property name="list" ref="bookList"></property>
</bean>

可以看出，这样就能把需要复用的list属性给提取了出来，然后用ref即可。
